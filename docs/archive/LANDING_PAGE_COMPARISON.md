## Problem Section

### Before
```
Headline: "Scientific interpretation is fragmented across tools, files, and expertise."

Problems:
1. Raw files - Instrument signals arrive across formats, folders, and naming conventions.
2. Scattered notes - Interpretation depends on context spread across notebooks and messages.
3. Repeated steps - Manual processing makes evidence review slow and inconsistent.
4. Disconnected tools - XRD, Raman, FTIR, and XPS work rarely share one reasoning path.
5. Hard to reproduce - Decisions need linked methods, caveats, confidence, and provenance.
```

### After
```
Headline: "Interpretation is the bottleneck, not data collection."

Problems:
1. Interpretation bottleneck - Data collection is fast. Turning signals into decisions is slow.
2. Scattered context - Evidence lives across notebooks, files, and conversations.
3. Manual reasoning - Each decision requires repeated interpretation and validation steps.
4. Disconnected techniques - XRD, XPS, FTIR, Raman insights rarely share one reasoning path.
5. Hard to reproduce - Scientific decisions need linked evidence, methods, and provenance.
```

**Key Changes**:
- ✅ Changed: Focus from "fragmented tools" to "interpretation bottleneck"
- ✅ Emphasized: Reasoning/interpretation as the problem, not data
- ✅ Clarified: "Disconnected techniques" (not just tools)

## Workflow Section

### Before
```
Title: "Agentic Scientific Workflow"

Steps (4):
1. Plan - Translate the scientific goal into a multi-technique workflow.
2. Execute - Process raw signals and detect technique-specific features.
3. Evidence - Link conclusions to spectra, methods, confidence, and caveats.
4. Report - Generate a traceable decision ready for notebook review.
```

### After
```
Title: "How It Works"
Headline: "Goal → Plan → Execute → Evidence → Reason → Decision → Report"

Steps (7):
1. Goal - Define the scientific question or material characterization objective.
2. Plan - Generate an executable workflow across relevant characterization techniques.
3. Execute - Run deterministic tools to process signals and extract structured features.
4. Evidence - Build evidence packets linking features, candidates, confidence, and uncertainty.
5. Reason - Apply reasoning layer over structured evidence to produce scientific conclusions.
6. Decision - Generate reproducible decisions with linked methods, caveats, and provenance.
7. Report - Output traceable reports ready for review, export, and integration.
```

**Key Changes**:
- ✅ Expanded: 4 steps → 7 steps (complete workflow)
- ✅ Added: "Goal" and "Reason" and "Decision" steps
- ✅ Clarified: Reasoning layer separate from execution
- ✅ Changed: Layout from horizontal flow to grid
